# CHANGELOG

## 6.0.0(2026.01.27)

-   新特性
    -   支持 zip 全量更新
        1.  插件会在构建后，为各个远程热更包自动生成 zip 文件
        2.  当某个热更包检查更新时，如果发现此热更包在本地没有历史记录时，插件会自动采用 zip 全量更新模式，下载热更包的 zip 文件
        3.  通过 zip 全量更新的方式，可以极大减少下载并发任务数、减少下载耗时、减少下载体积、减少 IO 操作
        4.  **当前 zip 全量更新模式，仅支持 Android 平台**，其他平台会自动回退会旧版本的差异更新模式
-   Breaking Changes
    -   为了适配新的热更新方式，API 发生了变更，详细参考[插件升级指南](https://www.yuque.com/dhunterstudio/gg/hot-update-migration-guide)

## 5.0.0(2025.12.16)

-   Breaking Changes
    -   调整各个热更包信息文件的输出路径
-   新特性
    -   **热更包支持将多个 AssetBundle 组合为一个热更包**
        -   **假设项目原始构建目录如下：**
            ```
            data/
            ┣━━ assets/
            ┃   ┣━━ internal/
            ┃   ┣━━ main/
            ┃   ┣━━ resources/
            ┃   ┣━━ bundle-game-a/
            ┃   ┣━━ bundle-game-b/
            ┃   ┣━━ bundle-game-c/
            ┃   ┗━━ bundle-game-d/
            ┣━━ jsb-adapter/
            ┣━━ src/
            ┣━━ application.js
            ┗━━ main.js
            ```
        -   **v5 版本之前，子包（子热更包）默认情况下都是只能配置一个 AssetBundle**:
            ```
            {
                "remote_bundles": {
                    "bundle-game-b": {
                        "files": [
                            "assets/bundle-game-b/"
                        ]
                    },
                }
            }
            ```
        -   **v5 版本以后，子包（子热更包）可以任意配置其所包含的内容，比如：一个子包包含多个 AssetBundle**
            ```
            {
                "remote_bundles": {
                    "bundle-game-b": {
                        "files": [
                            "assets/bundle-game-b/"
                        ]
                    },
                    "multi-bundle": {
                        "files": [
                            "assets/bundle-game-c/"
                            "assets/bundle-game-d/"
                        ]
                    },
                }
            }
            ```
-   优化
    -   完善 CocosCreator 构建面板上，插件相关配置参数的检验规则
    -   完善日志输出格式
    -   加入构建限制，如果构建时启用了 **构建面板** 的 **`MD5 缓存`** ，那么该次构建将会失败，并在编辑器控制台中输出提示
-   修复
    -   修复热更新资源时，如果出现下载失败，在再次触发热更新时，累计下载字节数(`downloadedBytes`)可能会比总字节数(`totalBytes`)大的问题

## 4.0.0(2025.09.11)

-   Breaking Changes
    -   移除 `ggHotUpdateManager.isHotUpdateBundle()` API
-   新增
    -   `ggHotUpdateManager.clear()`: 清空所有热更新数据。当游戏发生不兼容更新时（比如升级了引擎），在新版本应用覆盖安装后可以调用此方法清空所有热更新数据并重启游戏
-   优化
    -   优化所有热更包（主包/子包）的检查更新逻辑：移除 `fetch()` API 的使用，改为采用下载 `version.manifest` 文件的方式去获取远端版本做版本比较
        -   这是因为 `fetch()` 在不同引擎版本的不同原生平台的实现存在差异，部分引擎版本通过此 api 获取信息是存在问题(如 3.6.3 iOS 平台)
-   修复
    -   修复特殊情况下，可能会将未热更新完毕的主包，从下载目录移动到搜索路径目录的问题

## 3.0.0(2025.02.10)

-   Breaking Changes
    -   移除版本号概念，热更新文件的差异将由插件完全托管，只要构建后的文件有差异，插件必定能识别，开发者不用再关心热更新文件的版本大小关系。**此变动将影响热更新构建文件的配置项**
        -   **v3 版本以前，每个主包/分包需要主动配置 `version` 字段以决定主包/子包的版本关系**:
            ```
            {
                "local_bundles": {
                    "build-in": {
                        "version": "0.0.0",
                        "files": ["src/", "jsb-adapter/", "assets/"]
                    }
                },
                "remote_bundles": {
                    "build-in": {
                        "version": "1.0.0",
                        "files": ["src/", "jsb-adapter/", "assets/"]
                    },
                }
            }
            ```
        -   **v3 版本以后，开发者只需要告诉插件主包/分包是如何划分即可，不再需要手动填入版本号:**
            ```
            {
                "local_bundles": {
                    "build-in": {
                        "files": ["src/", "jsb-adapter/", "assets/"]
                    }
                },
                "remote_bundles": {
                    "build-in": {
                        "files": ["src/", "jsb-adapter/", "assets/"]
                    },
                }
            }
            ```
    -   优化插件日志代码，只有构建时，勾选了「调试模式」时，日志代码才会存在，否则日志代码会被裁剪
-   Bugfix
    -   修复热更新资源下载到本地后，在移动资源文件到搜索路径后，删除下载缓存文件夹时，因为文件夹可能不存在，导致出现警告日志的问题
    -   修复热更新资源时，如果出现下载失败，在再次触发热更新时，累计下载字节数可能会比总字节数大的问题
    -   修复 ios 平台上，下载文件时，没有提前创建好文件本地存储地址的所有父目录，从而导致下载失败的问题

## 2.0.0(2024.11.07)

-   Breaking Changes
    -   构建任务选项 **「移除热更新远程包的输出目录路径」** ，此后所有「热更新远程包的输出目录」将会在「当前构建任务的输出根目录下的 data-gg-hot-update 目录」
    -   热更新远程包的输出目录的 zip 压缩包路径变更。以 Android 平台为例：
        -   插件 v1.x 版本：build/android/**`data-gg-hot-update/gg-hot-update-remote-assets.zip`**
        -   插件 v2.x 版本：build/android/**`data-gg-hot-update.zip`**
    -   为了方便**实现回滚**功能，我们将调整新版本判断逻辑变更。具体如下：
        -   插件 v1.x 版本：远程版本必须大于本地版本，才会回调发现新版本
        -   插件 v2.x 版本：远程版本不等于本地版本，且存在差异文件需要下载，就会回调发现新版本
